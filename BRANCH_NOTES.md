# 分支说明

这个项目目前建议长期保持两个主要分支。

## 1. `main`

定位：

- 稳定版
- 给同事使用
- 适合打包发版

当前建议包含的功能：

- 照片下载与分类
- 证件资料筛选
- 表样转换

版本建议：

- 当前稳定版建议使用 `v1.1.0`

使用场景：

- 给同事打包
- 给客户演示
- 日常正式处理数据

## 2. `codex/dev`

定位：

- 激进开发版
- 你自己继续加功能、试规则、做验证

当前建议包含的功能：

- `main` 中的全部功能
- 数据匹配
- 考场编排
- 结果打包
- 其他还在试验中的增强功能

使用场景：

- 新需求开发
- 复杂规则验证
- 新功能联调

## 推荐工作方式

### 日常开发

平时新增功能、改规则、试复杂需求时：

```bash
git checkout codex/dev
```

### 给同事发稳定版

准备给同事打包或发正式版本时：

```bash
git checkout main
```

### 当某个功能稳定后

先在 `codex/dev` 里测试稳定，再合并到 `main`：

```bash
git checkout main
git merge codex/dev
```

如果不想一次把全部开发内容都并到稳定版，也可以只挑需要的提交单独合并。

## 版本建议

建议按下面的方式走版本：

- `v1.1.0`：稳定版新增一组完整功能
- `v1.1.1`：小修复、小优化
- `v1.2.0`：下一轮新的稳定功能发布

不要按“增加了几个功能”直接跳版本号，而是按“发布轮次”升级。

## 建议

- `main` 尽量少直接改
- 新需求优先放到 `codex/dev`
- 确认稳定后再决定哪些功能进入 `main`
